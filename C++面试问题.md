### const有什么用途

1. 定义常量
2. 修饰函数参数和返回值
3. 修饰函数的定义体，这里函数为类成员函数，被const修饰的成员函数代表不修改成员变量的值

### 指针和引用的区别

1. 引用只是变量的一个别名，内部实现是只读指针
2. 引用只能在初始化时被赋值，其他时候不能被改变，指针的值在任何时候都能被改变
3. 引用不能为NULL,指针可以为NULL
4. 引用变量内存单元保存的是被引用变量的地址
5. “sizeof引用”=指向变量的大小，“sizeof指针”=指针本身大小
6. 引用可以地址操作哦，返回的是被引用变量本身所在内存单元地址
7. 引用的变量相当于普通变量使用，做函数参数时，内部传递的实际是内存地址

### malloc/free new/delete

1. malloc/free是c++/c的标准库函数，new/delete是c++的运算符。他们都可以申请动态内存和释放。
2. malloc/free无法完成对动态对象的内存管理，而需要new/delete，new除了动态分配内存还能构造对象，delete除了释放内存还能执行析构函数
3. malloc/free的功能new/delete都能实现，之所以没将malloc/free淘汰是因为c++经常会调用c程序，c程序中只能使用malloc/free释放内存

### 堆和栈的区别

1. 栈：存放函数参数值、局部变量，编译器自动释放，内存大小有限，空间连续，向地址减小方向生长，分配效率高
2. 堆：手动释放，内存大小没有限制，空间不连续，向内存地址增大方向生长，分配效率低

### 关键字static的作用

1. 函数体内static变量的作用范围为该函数体，不同于auto变量，该变量的内存只被分配了一次，因此该值保持不变
2. 模块内static全局变量可以被所有函数访问，但不能被模块外其他函数访问
3. 在模块内的static函数只被这一模块中其他函数调用，函数使用范围被限制在该模块中
4. 在类的static的成员变量属于整个类所有，对类的所有对象只有一份拷贝（唯一）
5. 在类中的static成员函数属于整个类所有，这个函数不接受tihs指针，只能访问类的static成员变量


### 在c++程序中调用被C编译器编译后的函数，为什么要加extern“C”

函数被c++ 编译器编译后在库中的名字与c语言不同。假设某个函数原型为：
```
void foo(int x,int y);
```
该函数被c编译器编译后在库中名字为：_foo<br/>
在c++ 编译器便后会产生像：_foo_int_int 之类的名字<br/>
如果不加extern"c" 就会搜索_foo_int_int从而找不到_foo()

### 线程和进程的联系和区别

1. 进程最重要的是并发性和动态性。
进程是程序的一次执行，可以与其他计算并发执行。
2. 线程是进程实施调度和分派的基本单位，操作系统提供线程的目的就是为了方便高效地实现并发处理。
3. 二者关系：
   1. 一个进程可以多个线程，但至少一个；一个线程只能在一个进程的地址空间内活动
   2. 资源分配给进程，即同一个进程的所有线程共线该进程的所有资源
   3. CPU分配给线程，即真正处理器运行的是线程
   4. 线程在执行过程中需要协作同步，不同进程的线程间要利用消息通信的办法实现同步
4. 进程时最基本的资源拥有单位和调度单位，进程之间的通信方式：消息传递、共享存储、管道通信

### 多线程同步和互斥有何异同

1. 线程同步是线程之间的一种制约关系，一个线程的执行以来另一个线程的消息；当它没有得到另一个线程消息是应一直等待，直到消息达到时被唤醒
2. 线程互斥是指对于共享进程系统资源，在各个单线程访问时的排他性。若干线程都要使用某一公共资源时，任何时候最多一个线程去使用，其他线程必须等待。

### 结构体和类的区别

C++中的struct对C的struct进行了扩充，struct可以包含成员函数，可以继承，可以实现多态
但是struct和class一个本质区别是：**默认访问权限不同，struct是public，class是private**，此外class还可以用来表示模板类型，struct不行
